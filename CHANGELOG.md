## 2026-07-20 · 인증 보안 강화(H2·H3·H4·L1)
- **관리자 세션 단축(H3)**: 관리자 세션·쿠키 수명을 12시간으로 단축(일반 회원은 7일 유지). 관리자 계정이 탈취되면 전 회원 개인정보·결제 상태에 접근할 수 있어 노출 시간을 줄임. 관리자 판별은 `ADMIN_EMAILS` 기준 — ⚠️ 값이 비거나 오타면 오류 없이 7일로 되돌아가므로 운영 배포 전 확인 필요
- **역할 변경 시 세션 회전(H4)**: 계정 역할이 바뀌면 기존 세션을 전부 무효화(`DELETE FROM auth_sessions`). 의사 계정이 병원으로 전환될 때 예전 세션이 새 권한을 그대로 얻던 문제 차단
- **계정 열거 차단(H2)**: 잠긴 계정에만 429를 돌려주던 것을 제거 — 아무 비밀번호나 5번 넣어보고 상태 코드만으로 가입 여부를 알 수 있었음. 이제 잠금 여부와 무관하게 해시를 계산한 뒤 동일한 401·동일 문구 반환(응답 시간 차이도 축소). 잠금 중 실패 횟수를 올리지 않아 잠금 무한 연장도 방지
- **보안 헤더(L1)**: HTML 응답에 CSP, HSTS, `X-Content-Type-Options`, `X-Frame-Options: DENY`, `Referrer-Policy`, `Permissions-Policy` 적용
  - ⚠️ CSP는 이니시스 결제창(`stdpay.inicis.com`, `stgstdpay.inicis.com`)을 `script-src`/`connect-src`/`frame-src`/`form-action`에 허용 중. **이 도메인을 빼면 결제창이 조용히 안 뜬다**(브라우저 콘솔에만 로그, 서버 로그·테스트엔 흔적 없음)
