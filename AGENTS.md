# AGENTS.md

## Purpose

AI 개발 도구가 메디헬퍼스의 사업 방향과 안전 기준을 유지하며 작업하기 위한 지침입니다.

## Current State

현재는 시장 검증용 프론트엔드 MVP입니다. 실제 인증·DB·결제는 문서에 정의되어 있으며 순차 도입합니다.

## Current Rules

- 모든 작업 전 `docs/INDEX.md`와 관련 주제 문서를 먼저 읽습니다.
- 메디헬퍼스의 핵심은 단순 공고 게시판이 아닌 1:1 의료 커리어 컨시어지입니다.
- 타 서비스의 상표, 문구, 화면을 복제하지 않습니다.
- 실제 의사 개인정보와 병원 내부정보를 샘플 데이터에 넣지 않습니다.
- 결제는 승인된 PG와 서버 검증 없이 성공 처리하지 않습니다.
- 개인정보 수집 폼에는 목적, 보유기간, 동의 체크를 포함합니다.
- 변경 후 `npm run build`를 실행합니다.
- 배포 시 `.openai/hosting.json`의 기존 `project_id`를 유지합니다.

## Brand Adaptation Rule

- 사용자가 경쟁 서비스나 참고 화면을 제공해도 색상, 테두리, 간격, 카드 모양을 그대로 복제하지 않습니다.
- 레퍼런스에서는 정보 구조, 사용자 흐름, 필요한 입력 항목과 상호작용만 추출합니다.
- 최종 화면은 메디헬퍼스의 Navy `#10213F`, Blue `#1263E8`, Mint `#20B894`, 밝은 Surface 색상과 충분한 여백을 사용해 독자적으로 재설계합니다.
- 두껍고 검은 외곽선, 과도한 장식, 작은 글자처럼 메디헬퍼스 톤과 맞지 않는 요소는 참고 화면에 있더라도 사용하지 않습니다.
- 주요 버튼은 충분히 크고 명확하게 배치하며, 폼과 정보량이 많아질수록 콘텐츠 폭·글자 크기·입력 높이를 함께 확장합니다.
- 구현 전 항상 “참고한 구조”와 “메디헬퍼스 방식으로 재해석할 시각 표현”을 분리해서 판단합니다.
