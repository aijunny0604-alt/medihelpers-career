# 이력서 파이프라인 통합 기획 (서버저장 + 파일업로드 + 자동첨부 + 관리자 열람)

기준일: 2026-07-18
관련 문서: `AUTH.md`, `SECURITY_DATA_EXPOSURE_PLAN.md`, `HEADHUNTING_BOARD_PLAN.md`, `API.md`

## 배경 (발견된 반쪽 기능들)

의사 후보 정보를 받는 통로가 3개인데 **모두 서버 저장이 안 됨**:
1. **이력서 등록(`/resume`)** — `medihelpers_resumes`로 localStorage에만 저장(ResumePage.jsx:120). 읽는 곳 0개. 아빠도 못 봄.
2. **헤드헌팅 상담(`/headhunting`) 파일 업로드** — "기존 이력서 공유"(PDF·DOC·DOCX·HWP·HWPX ≤10MB) UI는 있으나(HeadHunterRequestPage.jsx:375~392), 제출이 `JSON.stringify(payload)`라 **파일이 안 실려 버려짐**(98~102행).
3. **상담 텍스트** — 이건 서버 저장됨(`/api/consultations`, 정상). 파일만 누락.

즉 의사가 아무리 이력서를 쓰거나 파일을 올려도 **아빠에게 도달하지 않는다.**

## 전략 결정 (2026-07-18 사용자 확정)

### 우리 홈 이력서 "전용"으로 간다 ✅ (외부 파일 업로드 제거)
- **우리 홈 이력서(`/resume`)만 사용.** 외부 PDF/DOC/HWP 업로드는 제거.
- 이유:
  1. **프로그램적으로 훨씬 간편** — R2 파일저장소·악성파일 검사·presigned URL·비공개버킷이 전부 불필요. 개발량 절반 이하, 보안 위험 제거.
  2. **데이터 100% 구조화** — 진료과·경력·희망지역이 필드로 저장 → 검색·필터·매칭·인재정보란 자동등록 가능. PDF는 검색 불가.
  3. **후보 이탈 우려 낮음** — 아빠가 큐레이션하는 헤드헌팅(대량지원 아님)이라, 진지한 이직 의사는 우리 양식 작성 부담이 크지 않음.
- **조치**: 헤드헌팅 상담 폼의 "기존 이력서 공유" 파일 업로드 UI(HeadHunterRequestPage.jsx:375~392)는 제거하고, "이력서 작성하기(`/resume`)" 링크로 대체.

## 목표 흐름

```
의사 → [우리 이력서 작성] 또는 [외부 파일 업로드]
        → 서버 저장 (이력서=D1 구조화 / 파일=R2)
        → 아빠 관리자 콘솔에서 열람·다운로드
        → (동의 시) 인재정보란에 익명 등록  [[HEADHUNTING_BOARD_PLAN]]
병원 → 후보 보고 "연결 요청" → 그 의사의 이력서/파일이 케이스에 자동 첨부
```

## 구현 범위 (M단위) — 파일 업로드(R2) 제거로 단순화

1. **M1 · 이력서 서버 저장** — `/resume` 제출을 localStorage에서 `POST /api/resumes`로 전환. D1 `resumes` 테이블(account_id, 구조화 필드, 공개동의, 상태). 로그인 의사 본인 것만. **(핵심)**
2. **M2 · 파일 업로드 UI 제거** — 헤드헌팅 상담 폼의 "기존 이력서 공유" 파일 필드 제거 → "이력서 작성하기" 유도 링크로 대체. (R2 연동 자체가 불필요해짐)
3. **M3 · 관리자 열람** — 관리자 콘솔에 이력서 목록/상세. 열람 감사 로그(`access_audit_logs`). 파일 다운로드 없이 구조화 데이터로 표시.
4. **M4 · 자동 첨부** — 상담/후보연결 요청 시 해당 의사의 최신 이력서(구조화 데이터)를 케이스(`recruitment_cases`)에 연결. 병원 "연결 요청" 원클릭과 연동. [[NOTIFICATION_SETUP]]
5. **M5 · 인재정보 연동(선택)** — 아빠가 이력서 검토 후 익명 인재 프로필로 등록(동의 기반). 이력서가 구조화돼 있어 필드 매핑만으로 등록 가능.

## 보안·법무 주의

- 이력서는 **개인정보**. 수집 목적·보유기간·동의 필수(LEGAL_PRIVACY_SIGNUP 준수).
- 회원 본인·관리자만 접근. 열람 감사 로그.
- 실명·연락처 포함 이력서는 인재정보란(익명)과 분리 저장. [[SECURITY_DATA_EXPOSURE_PLAN]]
- 파일 업로드 제거로 악성파일·저장소 보안 리스크 원천 제거됨.
