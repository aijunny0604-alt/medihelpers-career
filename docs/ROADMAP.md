# ROADMAP

## Purpose

시장 검증부터 실제 결제 플랫폼까지의 출시 순서를 정의합니다.

## Current State

### Phase 1 — 전환형 웹사이트

- [완료] 메뉴별 페이지, 채용 검색, 관심공고
- [완료] 의료인·병원 단계형 상담 신청
- [완료] 광고 상품·가격·주문 요청 화면
- [완료] 모바일·태블릿·데스크톱 반응형
- [완료] 8개 의료직군 통합 허브와 직군별 오픈 알림 진입
- [완료] 의사 이직 전담상담 페이지와 직접 전화·상담 접수 전환 흐름
- [다음] 실제 사용자 기반 접근성·가독성 테스트와 SEO 메타데이터 정교화

### Phase 2 — 운영 백오피스

- Supabase DB와 관리자 로그인
- 상담 리드, 병원, 의료인, 공고 관리
- 신청 이메일·알림
- 채용공고 등록/검수/게시
- 직군별 파트너·오픈 알림 대기자 관리

### Phase 3 — 회원 플랫폼

- 의료인·병원 계정 및 면허·자격 인증
- 이력서/병원 서류 업로드
- 비공개 매칭과 지원 현황
- 찜·알림·마이페이지
- 직군·전문영역별 인증 커뮤니티

### Phase 4 — 결제와 성장

- 토스페이먼츠 결제창·웹훅·환불
- 광고 자동 게시와 기간 관리
- 세금계산서/영수증 처리
- 광고 성과 리포트와 재구매

## Current Rules

Phase 2의 운영 도구 없이 공개 회원가입과 자동 결제를 먼저 열지 않습니다.

직군 확장은 `의사 → 간호 → 방사선·임상병리 → 재활 → 약무·치과·응급보건` 순서를 기본으로 하며, 실제 병원 공고와 의료인 파트너 확보량을 기준으로 조정합니다.
