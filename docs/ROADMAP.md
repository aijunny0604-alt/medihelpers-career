# MEDIHELPERS PRODUCT ROADMAP

## 사업 우선순위

1. 병원 의사 헤드헌팅 성공보수
2. 병원 의사 초빙광고
3. 병원 브랜드관·추천 노출
4. 별도 의료인 채용 허브의 공고 등록비
5. 병원의 의료인 인재검색 이용권
6. 선택형 의사 커리어 컨시어지

메인 브랜드는 의사 전문 헤드헌팅으로 유지한다. 간호사·간호조무사·방사선사·임상병리사·물리/작업치료사·치과위생사·병원 행정직은 `/medical-staff` 서브 허브에서 분리 운영한다.

## 의사 서비스 원칙

- 의사 인증 회원의 핵심 공고 정보와 헤드헌터 상담은 무료다.
- 급여 구조, 실제 근무표, 당직, 휴무, 진료 강도와 입사 판단 정보는 열람권으로 판매하지 않는다.
- 유료 서비스는 맞춤 공고 즉시 알림, 우선 상담 일정, 연봉·계약 분석, 정기 커리어 점검처럼 시간과 판단을 줄이는 선택 기능에 한정한다.
- 병원 후보 문의는 후보자 동의와 헤드헌터 확인을 거친다. 후보 열람권이나 소개권을 직접 판매하지 않는다.

## 헤드헌팅 CRM 단계

`신규 의뢰 → 조건 확인 → 후보 탐색 → 후보 동의 → 병원 제안 → 면접 예정 → 조건 협상 → 입사 확정 → 종료`

각 채용 건은 담당 헤드헌터, 병원 의뢰서, 후보 추천, 제3자 제공 동의, 면접 일정, 제안 조건, 입사일, 성공보수 청구 상태를 함께 보관한다.

## 권한·개인정보 원칙

- 공개 API에는 익명 후보 정보만 반환한다.
- 후보 실명과 연락처는 프런트엔드 정적 데이터에 넣지 않는다.
- 병원·관리자도 후보자의 병원별 공개 동의가 있어야 실명을 열람한다.
- 동의 철회 즉시 접근을 차단한다.
- 실명 열람, 후보 추천, 단계 변경, 동의 철회는 감사기록에 남긴다.
- 업로드 파일은 R2, 소유권·공개 범위·검수 상태는 D1에 저장한다.

## 단계별 출시

### Phase A — 현재 구축

- 헤드헌팅 중심 홈과 병원/의사 핵심 CTA
- 의사 인증 후 상세조건 무료 정책
- 선택형 커리어 서비스
- 병원·의사 마이페이지
- 상담 접수와 관리자 알림
- 채용 CRM 데이터 구조와 관리자 파이프라인
- 별도 의료인 채용 허브

### Phase B — 운영 연결

- 의사면허·병원 사업자 인증 공급자 연동
- 실제 후보 실명 보관소와 병원별 동의 워크플로
- 상담을 채용 건으로 전환하는 운영 버튼
- 면접 일정 이메일·문자 알림
- 성공보수 계약·청구·입금 상태
- 광고 검수·게시·기간·성과관리

### Phase C — 거래 시스템

- PG 결제와 현금영수증·세금계산서 처리
- 광고 자동 게시와 만료
- 의료인 공고·지원자 관리
- 의료인 이력서·인재검색 이용권
- 병원 채용 성과 리포트

## 공개 전 필수 조건

- 이용약관·개인정보처리방침·제3자 제공 동의·환불정책 법률 검토
- `SIGNUP_ENABLED`, `LEGAL_DOCUMENT_STATUS`, `ACCOUNT_HASH_SECRET` 운영값 설정
- 관리자 이메일 허용목록과 감사 로그 보존 정책
- 메일·문자 공급자 키, 발신번호, 수신자 설정
- 의사면허 및 병원 인증 방식 확정
- 성공보수 계약서와 광고 상품 계약 기준 확정
