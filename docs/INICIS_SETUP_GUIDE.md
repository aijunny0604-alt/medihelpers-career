# 이니시스 결제 설정 가이드 (실전환 절차)

> 코드는 **모두 구현 완료**되어 있습니다. 아래 3단계만 하면 실제 카드 결제가 작동합니다.
> (관련 설계: `docs/BILLING_INICIS_PLAN.md`, 구현: `src/inicisPay.js` + `scripts/package-sites.mjs`)

## 준비물 (이니시스에서 받는 것)

| 항목 | 설명 | 어디서 |
|---|---|---|
| **MID** | 상점 아이디 (예: `medihelp01`) | 이니시스 가맹점 관리자 / 영업담당 |
| **signKey** | 웹표준결제 사인키 (긴 문자열) | 이니시스 상점관리자 → 결제연동 정보 |
| **결제 도메인 등록** | 결제창을 띄울 도메인 | 이니시스에 **신청** 필요 |

⚠️ **signKey는 절대 깃에 커밋하거나 프론트엔드에 넣지 마세요.** 호스팅 환경변수(시크릿)로만 넣습니다.

---

## 1단계 — 도메인 (✅ 2026-07-20 결정: 기존 도메인 그대로 사용)

**결정 사항: 기존 `medihelpers.co.kr` 도메인을 새 사이트로 옮겨 사용한다.** (병행 운영 아님)

이 방식이면 이니시스 쪽에서 할 일이 **없습니다**:

| 항목 | 상태 |
|---|---|
| MID | ✅ 기존 것 그대로 |
| signKey | ✅ 기존 것 그대로 |
| 결제 도메인 등록 | ✅ **이미 등록돼 있음**(같은 도메인) |
| 이니시스 문의·신청 | ✅ 불필요 |
| 추가 계약·비용 | ✅ 없음 |

### 진행 순서
1. 새 사이트 완성 → 배포
2. `medihelpers.co.kr` DNS를 새 사이트(OpenAI Sites)로 연결
3. 환경변수 3개 설정(2단계) → 결제 즉시 작동

⚠️ **주의**
- 도메인을 옮기는 순간 **기존(구) 사이트는 접속 불가**가 됩니다. 새 사이트가 완전히 준비된 뒤에 옮기세요.
- 기존 사이트의 회원·공고 등 **중요 데이터는 미리 백업/이전** 여부를 확인하세요.
- 결제창은 **이니시스에 등록된 도메인에서만** 열립니다. 훗날 다른 도메인으로 바꾸면 **이니시스에 변경 신고**가 필요합니다.

### (참고) 만약 병행 운영으로 바뀐다면
기존 사이트와 새 사이트를 **둘 다** 살릴 경우에만 이니시스에 문의가 필요합니다:
> "현재 MID에 **도메인 추가 등록**이 가능한가요? 불가하면 새 MID를 발급받아야 하나요?"

---

## 2단계 — 환경변수 3개 설정 (호스팅 시크릿)

OpenAI Sites 호스팅 환경변수에 아래 3개를 넣습니다.

| 변수명 | 값 | 예시 |
|---|---|---|
| `INICIS_MID` | 이니시스 상점 아이디 | `medihelp01` |
| `INICIS_SIGN_KEY` | 웹표준결제 사인키 | `SU5JQ0lTX1RFU1RTSUdOS0VZ...` |
| `SITE_ORIGIN` | 사이트 주소(끝에 `/` 없이) | `https://medihelpers.co.kr` |

**선택 변수**

| 변수명 | 용도 |
|---|---|
| `PAYMENT_LIVE=true` | 실결제 강제. 이 값이 true인데 키가 없으면 결제를 **막습니다**(키 누락으로 전 주문이 무료 통과되는 사고 방지) |

### 설정하면 자동으로 일어나는 일
- 주문 생성 응답에 `inicis.configured=true`가 내려옴
- 프론트가 **실제 이니시스 결제창**을 띄움 (그 전까지는 테스트 가상결제)
- **코드 수정 불필요**

---

## 3단계 — 실결제 테스트

⚠️ **localhost에서는 테스트 불가**입니다. 이니시스가 결제 결과를 서버(returnUrl)로 보내야 하는데, 로컬 주소로는 못 보냅니다. **배포된 도메인에서만** 가능합니다.

### 테스트 순서
1. 배포 후 `https://<도메인>/membership` 접속 → 멤버십 결제 시도
2. 이니시스 결제창이 뜨는지 확인
3. **테스트 MID면** 이니시스 테스트 카드로 결제
4. 결제 후 `/mypage?payment=paid`로 돌아오며 **"결제가 완료되었습니다"** 배너 표시 확인
5. 결제창을 그냥 닫아보고 → **"결제가 완료되지 않았습니다"** 배너 확인
6. 관리자 콘솔 → 결제·환불 관리에서 주문이 `paid`로 기록됐는지 확인

### 결제가 붙는 3곳
| 위치 | 상품 |
|---|---|
| `/membership` | 멤버십 |
| `/advertise` → 광고 신청 | 병원 채용 광고(베이직·추천·집중) |
| 인재 열람권 구매 | 구직 의료인 이력서 열람권 |

## 문제가 생기면

| 증상 | 원인 | 조치 |
|---|---|---|
| 결제창이 안 뜸 | 도메인 미등록 / 키 오타 | 이니시스에 도메인 등록 확인, 환경변수 재확인 |
| 결제 후 JSON 화면이 뜸 | (수정됨) 서버가 결과 페이지로 리다이렉트 | 최신 배포본인지 확인 |
| "결제 설정이 완료되지 않았습니다" | `PAYMENT_LIVE=true`인데 키 없음 | 키를 넣거나 `PAYMENT_LIVE` 제거 |
| 금액이 안 맞다고 거절 | 주문 금액과 승인 금액 불일치(위변조 방어) | 정상 동작. 상품 가격 변경 중이면 재시도 |

## 아직 구현 안 된 것 (필요 시 추가)

- **웹훅/노티 수신**: 이니시스가 보내는 결제 결과 통보를 별도 저장(`payment_webhook_events`)해 대사(reconcile)하는 기능
- **가상계좌·계좌이체**: 현재는 카드 결제만
- **정기결제(빌링키)**: 멤버십 자동 갱신이 필요해지면 별도 기획
