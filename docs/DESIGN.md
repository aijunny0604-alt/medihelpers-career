# DESIGN

## Purpose

메디헬퍼스만의 브랜드와 반복 UI 기준을 정의합니다.

## Current State

### Concept — Human-led Doctor Headhunting

대형 구인구직 포털처럼 정보량만 강조하지 않고, 의사 전담 헤드헌터가 진료과·보수·당직·진료범위·입사 시점을 끝까지 조율한다는 신뢰를 전면에 둡니다.

### Visual System

- Navy `#10213F`: 전문성·신뢰
- Blue `#1263E8`: 연결·주요 행동
- Mint `#20B894`: 케어·검증·성공
- Surface `#F5F8FC`: 복잡한 채용 정보를 편하게 읽는 배경
- 둥근 카드, 얇은 경계선, 넉넉한 여백을 사용합니다.

### Navigation

- 의사 초빙정보: 의사용 공개·비공개 공고 탐색
- 인재정보: 병원용 익명 의사 인재풀 소개
- 의사 헤드헌팅: 의사 이직/병원 채용 상담
- 광고센터: 병원용 광고 상품·주문
- 서비스 소개: 신뢰·절차·연락처
- 예전 `의사 이직상담`(`/professions`) 진입점은 헤드헌팅(`/headhunting`)으로 통합했습니다. 옛 경로는 base 경로를 보존하는 SPA 리다이렉트로 연결합니다.

### 헤드헌팅 간편 접수 폼

- 병원은 `의사 초빙 의뢰서`, 의사는 `의사 선생님 간편 이력서`라는 사용자 언어를 사용합니다.
- 최소 필수정보와 선택정보를 분리하며 연령·성별 등 민감하거나 차별로 이어질 수 있는 조건은 필수로 요구하지 않습니다.
- 상단에 담당 연락처와 필수표시 안내, 하단에 수집 목적·파기 시점과 필수 동의를 함께 배치합니다.
- 레거시 종이 양식의 두꺼운 검은 외곽선은 사용하지 않고 Blue·Mint 포인트, 밝은 Surface, 16px 입력 글자와 54px 이상 입력 높이를 적용합니다.

### 운영 회원가입

- 자체 이메일·비밀번호 가입 후 일반·병원 회원 유형과 필수 동의를 서버 계정에 기록합니다. 비밀번호 원문은 저장하지 않습니다.
- 넓은 셸(최대 ~1340px)과 큰 히어로 제목(clamp 46~64px), 본문 18~20px, 입력 필드 높이 58px를 사용합니다.
- 데스크톱은 2열 입력 그리드(병원은 기관명·담당 역할 필드 포함), 768px 이하에서 1열로 접히고 360/390px 폭에서도 가로 스크롤이 생기지 않습니다.
- 필드 오류는 `aria-invalid`/`aria-describedby`와 초점 테두리로 표시하고, 완료·초기화 시 모든 입력·동의를 즉시 비웁니다.

#### 반응형 레이아웃 스케일

- 큰 화면일수록 여백·타이포를 키우는 프리미엄 스케일을 적용합니다: 셸 최대 ~1340px, 히어로 제목 clamp 46~64px, 본문 18~20px, 입력 필드 높이 58px.
- 입력 그리드(`.signup-field-grid`)는 데스크톱에서 2열(`grid-template-columns:1fr 1fr`), **768px(≤780px) 이하에서 1열**로 접힙니다. 1열 전환은 첫 파티 가입 base 블록 **뒤에** 오는 반응형 오버라이드로 강제해, 나중에 선언된 2열 base 규칙이 모바일을 덮어쓰지 못하게 합니다(캐스케이드 순서 고정).
- 360/390px 등 좁은 폭에서도 가로 스크롤이 생기지 않도록 `min-width:0`과 유동 폭을 사용합니다.

#### 개인정보 최소화 경계

- 가입 단계에서는 로그인 이메일, 비밀번호 해시, 최소 프로필, 회원 유형과 필수 동의 기록을 처리합니다.
- 면허·자격·기관 증빙은 가입과 분리하고 실제 기능 사용 시 필요한 범위에서만 확인합니다.
- 실제 계정 생성과 인증은 서버에서만 처리하며, 비밀번호 원문은 전송 후 저장하지 않고 즉시 단방향 해시로 변환합니다.

### Interaction

- 모든 주요 버튼은 실제 페이지, 전화, 이메일 또는 신청 흐름으로 연결합니다.
- 채용공고 카드는 빈 공간을 포함한 카드 전체를 상세보기 클릭 영역으로 사용하고, 관심공고 버튼만 독립 동작으로 분리합니다.
- 카드 전체 클릭은 키보드 초점과 Enter·Space 입력을 지원하며 명확한 초점 테두리를 표시합니다.
- 모바일에서는 메뉴를 접고 하단에 상담 전환을 유지합니다.
- 성공 메시지는 사용자가 무엇을 제출했는지 구체적으로 알려줍니다.
- 스크롤 진입 요소는 짧은 흐림과 아래 방향 이동에서 감속하며 정착하는 관성형 모션을 사용합니다.
- 첫 화면에만 약한 시차 효과를 적용하고, 스크롤 진행 상태는 상단의 얇은 선으로 표시합니다.
- PC의 마우스 휠·트랙패드 스크롤에는 짧은 관성과 감속을 적용해 페이지 자체가 부드럽게 따라오게 합니다.
- 모바일 터치 스크롤과 모달 내부 스크롤은 운영체제의 기본 동작을 유지합니다.
- `prefers-reduced-motion` 환경에서는 스크롤 등장·시차·진행 애니메이션을 비활성화합니다.
- Windows 화면 배율과 GPU 차이에서도 글자가 선명하도록 카드에는 블러·상시 3D 변환·확대 변환을 적용하지 않습니다.
- 공식 로고 사용권이 확인되지 않은 샘플 공고에는 생성형 광고 디자인 마크임을 명시하고 공식 로고처럼 오인되지 않게 합니다.
- 카드 스크롤 모션은 블러·확대 없이 22px 수직 이동과 페이드만 사용하고, 호버는 4px 이동과 광원·그림자로 제한합니다.
- 페이지 이동은 현재 본문이 170ms 동안 살짝 올라가며 사라지고 새 본문이 560ms 동안 감속해 정착하도록 하며 블러는 사용하지 않습니다.

### Simplicity Rules

- 첫 화면의 핵심 행동은 `채용 찾기 / 비공개 상담 / 공고 등록` 세 가지로 제한합니다.
- 장식용 숫자와 사실 근거가 약한 통계 띠는 사용하지 않습니다.
- 카드에 표시하는 조건은 진료과, 지역, 근무일, 급여 순으로 고정합니다.
- 광고센터는 상단과 모바일 하단에 항상 노출하되 일반 채용공고와 혼동되지 않게 표시합니다.
- 광고 공고는 별도 광고 등급으로 관리하며, 로고가 등록된 경우 로고를 카드의 중심 브랜드 요소로 크게 표시합니다.
- 로고가 없는 광고 공고는 병원명을 워드마크처럼 강조하고, 추천·집중채용 등급에 따라 배경과 테두리의 강도를 구분합니다.
- 광고 카드의 빛 효과는 스크롤 진입 시 한 번만 실행하고 반복 재생하지 않습니다.
- 집중채용은 목록 최상단과 가장 강한 1회성 포커스 모션을, 추천 광고는 다음 순위와 중간 강도 모션을 사용합니다.
- 광고센터의 가상 병원 배너는 디자인 예시 · 실제 공고 아님을 고정 표시하며 실제 채용 목록 데이터로 사용하지 않습니다.
- PC 광고 카드 호버는 포인터를 따라오는 미세 광원·1도 미만 기울기·느린 감속을 사용하고 터치·모션 감소 환경에서는 비활성화합니다.
- 광고센터 라이브 프리뷰와 실제 채용 목록은 같은 카드 구조를 사용해 광고주가 신청 전에 최종 노출 형태를 그대로 확인할 수 있게 합니다.
- 의사 이외 직군 확장 UI는 노출하지 않고, 의사 이직 전담상담 페이지에 상담 대상·준비 조건·진행 절차·직접 전화·접수 폼을 한 흐름으로 배치합니다.

### 채용정보 균형 노출 (Balanced Exposure)

- 채용정보 목록은 `진료과·지역 균형순`으로 노출합니다. 프리미엄(집중채용·추천) 광고는 최상단 프리미엄 영역에 그대로 유지하되, 프리미엄이 많아도 초기 **6건**만 보여주고 `프리미엄 공고 더보기`(남은 개수 표시)로 확장해 일반 목록을 끝없이 밀어내지 않습니다.
- 일반 공고는 진료과·지역이 번갈아 나오도록 균형 배치하고 초기 **9건**만 노출한 뒤 전체 폭 `공고 더보기`(남은 개수 표시)로 9건씩 확장합니다. 노출/전체 개수를 `9/24`처럼 명확히 표기합니다.
- 목록 상단의 진료과 스트립은 `전체`와 실제 진료과별 개수 배지를 보여주고, 모바일에서는 가로 스크롤합니다. 칩은 선택 상태(`aria-pressed`)와 44px 이상 터치 영역, `focus-visible` 초점을 가집니다.
- `균형 노출` 설명 패널은 "프리미엄은 최상단 유지, 아래 목록은 진료과·지역을 번갈아 노출"을 간결히 안내하며, 무작위 순위나 유료 노출 숨김을 암시하지 않습니다(세로 공간 최소화).
- 순서는 하루 단위 결정적 회전 seed로 고정해 같은 날에는 안정적이고, 유료 최상단 약속은 그대로 지킵니다.

## Current Rules

- 의사 사진을 장식 목적으로 남용하지 않습니다.
- 익명 인재카드는 식별 가능한 개인정보를 포함하지 않습니다.
- 애니메이션은 정보의 등장 순서를 돕는 범위에서만 사용하고 반복·회전·큰 시차 효과는 사용하지 않습니다.

## Reference Interpretation

- 외부 화면은 복제 대상이 아니라 문제 해결 방식과 정보 우선순위를 파악하기 위한 참고 자료입니다.
- 레이아웃을 참고하더라도 메디헬퍼스의 블루·네이비·민트 팔레트, 부드러운 라운드, 밝은 표면, 절제된 그림자로 다시 설계합니다.
- 검은색 굵은 테두리, 작은 본문, 좁은 입력창은 사용하지 않습니다. 데스크톱 주요 폼 본문은 16px 이상, 입력 높이는 52px 이상을 기본으로 합니다.
- 한 화면의 핵심 CTA는 중앙 또는 시선이 자연스럽게 도달하는 위치에 충분한 크기로 배치합니다.
- 광고센터 상단은 추상적인 기능 카드 대신 집중채용·추천·일반공고의 실제 노출 우선순위와 신청 행동을 한 화면에 보여줍니다.
- 매칭 리포트는 사람이나 기관의 절대 순위·단일 종합점수를 만들지 않고 `공개 확인 / 협의 필요 / 추가 확인`과 그 근거를 나란히 보여줍니다.
- 매칭 리포트와 멤버십은 다른 주요 페이지와 동일하게 데스크톱 최대 1,500px 콘텐츠 폭을 사용하고, 작은 화면에서는 16px 중심의 입력·본문과 한 열 카드 구조를 유지합니다.
- 병원 광고 이미지는 로고형이면 원본 비율을 유지한 대형 마크와 병원명을 함께 배치하고, 배너형이면 전체 폭을 채우는 `cover` 방식과 중앙 기준 크롭을 사용합니다.
- 배너형 광고는 카드 안의 작은 썸네일로 축소하지 않고 브랜드 스테이지 전체를 채우며, 병원명과 광고 등급은 반투명 오버레이로 분리해 이미지와 텍스트를 함께 선명하게 보여줍니다.
- 홈 검색 목록은 운영체제 기본 팝업 대신 최소 50px 행 높이의 키보드·마우스 대응 맞춤 목록을 사용합니다. 시스템 모션 축소 설정은 존중하되 사용자가 사이트에서만 명시적으로 다시 켤 수 있습니다.
- 홈 히어로 검색 패널(`hero-search-card`)은 단순 폼이 아니라 첫 화면의 주요 행동으로 읽히도록, 넓은 데스크톱(≥1400)에서 최대 약 1,080px 폭·큰 라운드·파란빛 테두리와 다층 그림자·56~58px 아이콘·큰 라벨/값·258px CTA를 사용합니다. 1101~1399 컴팩트 데스크톱은 폭·CTA·필드 높이를 살짝 줄여 컨시어지 카드와 균형을 유지하고, ≤780 반응형은 기존 세로 적층·최소 52px 컨트롤을 그대로 지킵니다.

## 프리미엄 광고 자동 순환

- 프리미엄 영역은 데스크톱 2개, 모바일 1개 단위로 노출한다.
- 같은 광고 등급 안에서 세션별 시작 순서를 달리하되, 등급 우선순위는 바꾸지 않는다.
- 광고 영역이 화면에 보이는 동안 7초마다 다음 묶음으로 전환한다.
- 마우스 오버, 키보드 포커스, 백그라운드 탭에서는 자동 전환을 멈춘다.
- 이전·다음 버튼과 페이지 표시를 제공하고 `prefers-reduced-motion`에서는 수동 탐색만 허용한다.

- 홈 검색은 외곽 카드 크기보다 실제 초빙유형·진료과·지역 컨트롤의 폭과 높이를 우선한다.
- 의사 초빙 검색은 `초빙유형 / 진료과 / 지역 / 근무조건` 정보 구조를 사용하고, 다른 의료직군 선택은 제공하지 않는다.
