# 현재 구현 상태

기준일: 2026-07-19

## 운영 중

- React·Vite SPA를 OpenAI Sites에 배포 (D1 런타임 API는 `scripts/package-sites.mjs`에 패키징)
- 홈·채용정보 공고 카드(등급별 색 통일, 제목·정보 가독성), 광고 공고는 급여 공개
- 의사 채용정보(/jobs, 병원 유료 광고 공고), 의료인 구인구직(/medical-staff, 구직 의료인 게시판+열람권), 맞춤 헤드헌팅(의사 초빙 정보란+상담)
- 일반/병원 회원가입, 역할별 마이페이지(프로필·알림·상담·결제·활동 D1 연동)
- 메디헬퍼스 자체 이메일·비밀번호 로그인(PBKDF2 해시, D1 세션, 7일 보안 쿠키, 로그인 실패 잠금)
- **회원 전용 접근 게이트**: 의료인 구인구직·이력서를 비로그인 시 블러+로그인 유도. 옛 경로는 `ROUTE_ALIASES` 동기 정규화(`/talent`→`/medical-staff`, `/professions`→`/headhunting`, `/advertise/post`→`/advertise`). 매칭 리포트 제거됨
- **이력서**: 서버(D1) 저장 → 관리자 콘솔 '이력서 관리' 열람 → 마이페이지 완성도 지표
- **결제 라이프사이클**: 이니시스 웹표준결제 **연동 완료**(서버 서명·승인 API + 클라이언트 표준결제창 `src/inicisPay.js`, 멤버십·광고·열람권 3개 지점). `INICIS_MID`/`INICIS_SIGN_KEY`/`SITE_ORIGIN` 환경변수만 넣으면 실결제 전환(코드 수정 불필요). 미설정 시 가상(테스트) 결제로 동작. ⚠️ 실결제 검증은 이니시스에 등록된 배포 도메인에서만 가능(localhost 불가),
  결제 완료→광고 노출기간 부여, 환불/취소→노출 회수, 병원 '내 공고' 이용현황, 관리자 서비스현황
- **인재 열람권(주 수익모델)**: 병원이 열람권 결제(1명 5,000/5명팩 20,000원=건당 4,000원, 30일)→`talent_unlocks` 권한 기록→`GET /api/talent-detail/:id`가 해당 병원에만 이력서 연락처·상세 공개(감사 로그). 미결제 시 잠금+구매버튼
- **의료인 구직 노출(opt-in)**: 초간편 이력서(직군 자유입력·3단계) 등록 시 '구직 공개' 선택한 경우에만 익명 인재로 노출. 의사·간호·의료기사·약무 모두 의료인 구인구직(/medical-staff) 구직 게시판에 카드형으로 표시(진료과·지역 필터+검색). 병원은 열람권 결제 시에만 이름·연락처·상세 열람(결제 완료 후 `?open=코드`로 상세 자동 오픈). 자동노출·관리자승인 아님
- **병원 채용공고 = 유료 광고만**: 무료 직접게시 폐지(/advertise/post→/advertise 정규화). 병원은 광고 상품(베이직·추천·집중) 결제로만 /jobs 게시. 헤드헌터 채용의뢰(/request/hiring 상담)는 유지. ⚠️ 서버 `job_create`(POST /api/member-center) API는 아직 살아있어 클라 우회 시 무료 draft 생성 가능 — 정책 완전 강제하려면 서버 API 폐기 필요(미완)
- **회원가입 간소화**: 휴대폰 인증(본인확인) 절차 제거 — 휴대폰은 연락용 일반 입력. 이메일+비밀번호 자체 로그인으로 본인 식별. '일반 회원' 명칭은 '의료인 회원'으로 통일(내부 role 키는 doctor 유지)
- **관심공고(찜)** 서버 저장(기기 간 동기화), **폼 자동 채움**(상담·결제·헤드헌팅에 회원정보 자동 입력)
- **급여·병원정보 유출방어**: 비인증자에게 공고 급여·식별정보를 서버에서 제거(클라 마스킹 아님)
- **인재 열람 방어**: 병원 일일 열람 상한(기본 30명)+단시간 폭주 탐지+감사 로그, 관리자 콘솔 '인재 열람 감사' 탭에서 병원별 열람량·이상 열람 확인(SECURITY_DATA_EXPOSURE_PLAN)
- 상담 접수 D1 저장, 관리자 상담함, 이메일(Resend)·문자(Solapi) 알림(키 설정 시 발송)
- 관리자 통합 콘솔, 채용 CRM(의뢰→후보→면접→입사), 병원 추천후보 마이페이지 연결

## 부분 구현 / 남은 작업

| 영역 | 현재 동작 | 운영 전 남은 작업 |
|---|---|---|
| 배포 | Sites 버전 191, 커밋 `f2512b7` 운영 배포. `/api/account`·`/api/resumes`·자체 로그인 API 실동작 확인 | 기능 변경 때 동일 검증 반복 |
| 결제 | 가상 결제 + 이니시스 승인 API 골격 | **이니시스 MID/키 설정**, 클라 결제창(stdpay) 연결, 웹훅 수신 |
| 알림 | 메일·문자 코드 완성 | **Resend/Solapi 키 환경변수 설정**(docs/NOTIFICATION_SETUP.md) |
| 회원가입 | 역할·동의 기록 | 승인된 약관 버전, 본인확인, 면허·병원 인증 |
| 공고·인재 데이터 | 관리자 D1 + 정적 data.js 병합 | 정적 샘플을 D1로 이전(단일 원장화). 실제 공고·인재 데이터 입력 |
| 이미지/파일 | 프론트 선택·미리보기 | 파일 업로드는 제거(우리 이력서 전용, RESUME_PIPELINE_PLAN) |
| 법무 | 약관·개인정보 초안(현행이 구 사이트보다 최신) | **정식 오픈 전 법무·세무 검토** |

## 이번 최신화에서 확인·수정

- 상담 제출 시 가입된 계정의 역할과 상담 유형이 다르면 서버가 거부합니다.
- 상담 내역의 회원 연결 기준 이메일은 사용자가 임의 입력한 값이 아니라 인증된 계정 이메일로 고정했습니다.
- 손상된 상담·관리 콘텐츠 JSON 한 건 때문에 마이페이지나 관리자 콘솔 전체가 열리지 않도록 안전 파싱을 적용했습니다.
- 공개 카테고리 API의 캐시 헤더가 실제 응답에 반영되도록 응답 헬퍼를 수정했습니다.

## 알려진 핵심 차이

관리자 화면의 콘텐츠 건수는 D1에 등록된 운영 콘텐츠만 집계합니다. 공개 채용정보 화면은 D1 운영 콘텐츠와 `src/data.js`의 시연용 정적 데이터를 합쳐 보여주므로 현재 두 화면의 건수는 같지 않습니다. 운영 전 정적 데이터를 D1로 이전한 뒤 관리자 DB를 유일한 데이터 원장으로 바꿔야 합니다.
