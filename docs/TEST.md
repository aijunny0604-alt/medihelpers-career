# TEST

## Purpose

출시 전 반드시 확인할 사용자 흐름을 정의합니다.

## Current State

브라우저 저장 안전 처리와 최소 회원가입 입력 검증에 대한 자동 테스트 6건을 운영하며, 나머지 화면 흐름은 빌드와 수동 시나리오로 검증합니다.

## Core Scenarios

- 모든 상단·하단 메뉴가 올바른 페이지로 이동
- 채용 진료과·지역·키워드 필터 조합
- 관심공고 저장과 새로고침 후 유지
- 의료인/병원 상담 폼의 역할별 문구와 필수 동의
- 광고 상품 선택, 주문 요약, 결제 요청 완료
- 모바일 메뉴 열기·닫기와 작은 화면 폼 사용
- 키보드로 메뉴·모달·폼 조작
- 존재하지 않는 경로의 홈 복귀
- 회원가입 비활성 상태에서 개인정보를 수집하지 않는지 확인
- 인증 전 계정 생성 API 거부, 필수 역할·약관·연령·개인정보 안내 검증
- 의료인·병원 가입, 재접속 조회, 로그아웃, 계정 삭제
- DB에 이메일 원문·전화번호·면허번호·마케팅 동의가 저장되지 않는지 확인

## Current Rules

실제 결제 연동 후 성공·실패·취소·중복 웹훅·금액 변조 테스트를 추가합니다.

회원가입 공개 전 실제 Sites 인증 헤더, D1 마이그레이션, CSRF 방어, 기능 플래그와 약관 버전 기록을 통합 테스트합니다.
