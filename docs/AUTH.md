# AUTH

## 현재 로그인 구현

- 메디헬퍼스 자체 이메일·비밀번호 로그인을 사용합니다. OpenAI/ChatGPT 로그인으로 이동하지 않습니다.
- 비밀번호는 사용자별 무작위 salt와 PBKDF2-SHA-256(100,000회) 해시로 D1에 저장하며 원문은 저장하지 않습니다.
- 로그인 세션은 일반 회원 7일, **관리자 12시간** 만료입니다. 브라우저에는 `HttpOnly`, `Secure`, `SameSite=Lax` 쿠키만 저장하고 D1에는 세션 토큰의 SHA-256 해시만 저장합니다.
  - 관리자 판별은 `ADMIN_EMAILS` 기준이며, 세션 만료와 쿠키 `Max-Age`를 함께 맞춥니다. 관리자 계정이 탈취되면 전 회원 개인정보·결제 상태에 접근할 수 있어 노출 시간을 줄입니다.
  - ⚠️ `ADMIN_EMAILS`가 비어 있거나 오타가 나면 관리자도 7일 세션이 됩니다(오류 없이 조용히 완화됨). 운영 배포 전 반드시 값 확인.
- 로그인 5회 실패 시 해당 계정 로그인을 15분 잠급니다.
  - **잠금 상태는 응답 내용으로 구분되지 않습니다.** 잠긴 계정도 비밀번호 오류와 동일한 401·동일 문구를 반환하고, 잠금 여부와 무관하게 해시를 계산해 응답 시간 차이도 줄입니다(단, 완전한 상수 시간은 아님 — DB 조회 유무 등 잔여 편차는 남습니다). 예전에는 잠긴 계정에만 429를 돌려줘서 아무 비밀번호나 5번 넣어보면 가입 여부를 알 수 있었습니다(계정 열거).
  - 잠금 중에는 실패 횟수를 더 올리지 않습니다(공격자가 계속 시도해 잠금을 무한 연장하는 것을 방지).
- **역할이 바뀌면 해당 계정의 기존 세션을 전부 무효화**합니다(`DELETE FROM auth_sessions`). 의사 계정이 병원으로 전환될 때 예전 세션이 새 권한을 그대로 얻는 것을 막습니다.
- 기존 계정·역할·동의·프로필 데이터는 유지하고 `auth_credentials.account_id`로 자체 로그인 자격증명을 연결합니다.
- D1 테이블은 `ensureAccountSchema`가 런타임에 자동 생성하며 기존 D1 바인딩을 재생성하거나 다시 연결하지 않습니다.

## 역할

- General (`doctor` 내부 키): 공개 공고 탐색, 선택형 의료인 인증, 이력서, 비공개 상담 신청
- Hospital: 병원 인증, 광고 등록, 익명 인재 탐색, 유료 소개 요청
- Headhunter: 상담, 후보 추천, 동의 확인, 면접·조건 조율
- Admin: 회원·결제·권한·게시물·개인정보 열람 기록 관리

화면에는 `의료인 회원`으로 표시하지만 기존 DB와 API 호환을 위해 내부 역할 값은 당분간 `doctor`를 유지합니다. 이 값만으로 의사 면허가 인증된 것으로 판단하지 않습니다.

## 인증

- 의료인: 본인 확인과 면허 또는 자격 확인
- 병원: 사업자등록 정보와 담당자 재직 확인
- 운영자: 다중 인증과 최소 권한 원칙 적용
- Google·Kakao·Naver 로그인은 인증 수단으로만 사용하며, 권한과 자격은 서버 검증 후 별도로 부여합니다.
- 같은 이메일을 가진 소셜 계정을 자동 연결하지 않고 로그인된 계정에서 재인증 후 연결합니다.

## 운영 가입 게이트

운영 환경은 이용약관 v1.0·개인정보처리방침 v1.0, `SIGNUP_ENABLED=true`, `LEGAL_DOCUMENT_STATUS=approved`, 32자 이상의 `ACCOUNT_HASH_SECRET`이 모두 준비된 경우에만 실제 서버 계정 생성을 엽니다. 현재 Sites 운영 환경은 이 조건을 충족하며, 가입 동의에는 적용 문서 버전과 시각이 저장됩니다.

- 대상 경로: `/signup`(역할 선택), `/signup/doctor`, `/signup/hospital`.
- 운영 가입은 메디헬퍼스 자체 이메일·비밀번호 인증을 사용하며 비밀번호 원문은 저장하지 않습니다.
- 필수 동의: 서비스 이용약관 v1.0, 개인정보 수집·이용 v1.0, 만 14세 이상 확인. 마케팅 수신 동의는 가입과 분리합니다.
- 계정 DB에는 인증 이메일 원문 대신 서버 비밀키로 생성한 HMAC 계정 키, 회원 유형, 동의 종류·문서 버전·시각을 저장합니다.
- 의료인 자격과 병원 담당자 권한은 가입만으로 부여하지 않고 해당 기능을 사용할 때 별도 확인합니다.

## 운영 회원가입 구현

- `/signup/doctor` 의료인 회원과 `/signup/hospital` 병원 회원의 가입 경로·목적·가입 후 이동을 분리합니다.
- 현재 Sites에서 메디헬퍼스 자체 이메일·비밀번호 인증과 D1 세션을 사용합니다.
- 가입 입력은 `의료인 / 병원·의료기관` 역할 하나이며, 전화번호·면허번호·주민등록번호·마케팅 동의를 받지 않습니다.
- 이메일 원문은 자체 로그인 조회용 `auth_credentials`와 회원 운영 프로필에 저장하고, 기존 도메인 데이터 연결에는 HMAC-SHA-256 계정 키를 계속 사용합니다.
- 이용약관, 만 14세 이상 확인, 개인정보 처리 안내 확인을 서로 다른 유형과 문서 버전으로 기록합니다.
- 소스에 포함된 약관·개인정보 안내 버전이 `draft`가 아니고 `SIGNUP_ENABLED=true`, `LEGAL_DOCUMENT_STATUS=approved`, 32자 이상의 `ACCOUNT_HASH_SECRET`가 모두 설정된 경우에만 서버 가입 API를 엽니다.
- Google·Kakao·Naver 외부 로그인은 인증 공급자, 각 사업자 앱 심사와 키가 확정된 뒤 동일한 가입 완료 API에 연결합니다. 자격증명 없이 성공을 흉내 내는 버튼은 제공하지 않습니다.

### 의료인 회원

- 목적: 채용공고 탐색, 관심공고, 비공개 이직 상담, 이력서·지원 관리
- 가입 후 별도 확인: 비공개 공고 열람·소개 진행 시 의료인 면허 또는 자격
- 기본 이동: 채용정보
- 회원가입 시 휴대폰 본인인증 없음(휴대폰은 연락용 일반 입력), 이메일+비밀번호 자체 로그인

### 병원 회원

- 목적: 채용공고·구직 의료인 인재 탐색, 채용 의뢰, 열람권 결제, 동의 기반 소개 요청
- 가입 후 별도 확인: 공고 등록·소개 요청 시 의료기관과 담당자 관계
- 기본 이동: 의료인 채용(채용공고 + 구직 의료인 인재 통합)

## 권한 정책

- 무료 권한은 공개 공고와 익명 인재의 기본 조건, 헤드헌터 상담을 제공합니다.
- 실제 결제 완료 권한 발급은 아직 미구현이며 PG 연동 후 서버 `entitlement`로 관리합니다.
- 병원 후보 문의는 의료인의 연락처 구매가 아니라 헤드헌터 확인과 후보자 동의를 거치는 소개 요청입니다.
- 의료인의 신원 정보는 후보자의 명시적 동의를 확인한 소개 건에서만 필요한 범위로 공개합니다.
- 민감정보 열람, 다운로드, 소개, 권한 변경은 모두 감사 로그에 기록합니다.

- 처리방침과 소개 건별 제3자 제공 동의 문구를 처리 흐름 변경 때 함께 개정
- 유료 직업소개 또는 직업정보제공 사업 해당 여부 검토와 필요한 신고·등록
- 이용약관, 환불 정책, 자동결제 고지 확정

## 보안 헤더 (HTML 응답)

HTML 응답에 다음 헤더를 붙입니다. 생성 위치는 `scripts/package-sites.mjs`의 HTML 응답부입니다.

| 헤더 | 값 | 목적 |
|---|---|---|
| `Content-Security-Policy` | 아래 참고 | XSS·리소스 주입 차단 |
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` | HTTPS 강제 |
| `X-Content-Type-Options` | `nosniff` | MIME 스니핑 차단 |
| `X-Frame-Options` | `DENY` | 클릭재킹 차단 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 외부로 경로 유출 방지 |
| `Permissions-Policy` | 위치·마이크·카메라·payment 차단 | 불필요한 권한 차단 |

### ⚠️ CSP가 막을 수 있는 외부 리소스 (회귀 주의)

CSP는 허용 목록에 없는 외부 리소스를 **조용히** 차단한다. 서버 로그·자동 테스트에는 흔적이 남지 않고
브라우저 콘솔에만 기록되므로, **배포본을 실제 브라우저로 열어봐야** 발견된다.

현재 반드시 유지해야 하는 허용 항목:

| 리소스 | 지시어 | 빼면 생기는 일 |
|---|---|---|
| `https://fonts.googleapis.com` | `style-src` | **사이트 전체가 대체 글꼴로 렌더**(2026-07-22 실제 발생) |
| `https://fonts.gstatic.com` | `font-src` | 위와 동일(폰트 파일 자체가 차단) |
| `https://stdpay.inicis.com`<br>`https://stgstdpay.inicis.com` | `script-src`·`connect-src`·`frame-src`·`form-action` | 결제창이 뜨지 않음 |

이니시스 결제창의 경우 사용자에게는 스크립트 로드 실패 안내가 표시되지만(`src/inicisPay.js`),
글꼴 차단은 아무 안내 없이 화면만 바뀌므로 더 늦게 발견된다. CSP를 조이는 작업을 할 때 위 표를 반드시 확인할 것.
