# AUTH

## 역할

- Doctor: 공개 공고 탐색, 의료인 인증, 유료 공고 열람, 상담 신청
- Hospital: 병원 인증, 광고 등록, 익명 인재 탐색, 유료 소개 요청
- Headhunter: 상담, 후보 추천, 동의 확인, 면접·조건 조율
- Admin: 회원·결제·권한·게시물·개인정보 열람 기록 관리

## 인증

- 의료인: 본인 확인과 면허 또는 자격 확인
- 병원: 사업자등록 정보와 담당자 재직 확인
- 운영자: 다중 인증과 최소 권한 원칙 적용
- Google·Kakao·Naver 로그인은 인증 수단으로만 사용하며, 권한과 자격은 서버 검증 후 별도로 부여합니다.
- 같은 이메일을 가진 소셜 계정을 자동 연결하지 않고 로그인된 계정에서 재인증 후 연결합니다.

## 프론트엔드 전용 가입 신청 초안 폼 (정식 오픈 전)

정식 오픈 전에도 홈페이지에서 회원가입 양식을 직접 작성해볼 수 있도록, `SIGNUP_ENABLED=false`인 동안에는 오류·차단 게이트 대신 프론트엔드 전용 **가입 신청 초안(draft) 폼**을 제공합니다. 이 폼은 실제 계정을 만들지 않으며, 형식 확인까지만 처리합니다.

- 대상 경로: `/signup`(역할 선택), `/signup/doctor`, `/signup/hospital`.
- 공통 입력: 이름, 휴대폰 번호, 이메일, 비밀번호, 비밀번호 확인. 병원 회원은 병원·기관명과 담당자 역할을 추가로 입력합니다.
- 검증: 비밀번호는 영문·숫자를 포함한 8자 이상, 휴대폰은 한국 휴대폰 형식, 이메일 형식과 비밀번호 일치를 확인하며 한글 오류 메시지와 `aria-invalid`/`aria-describedby`로 접근성 있게 표시합니다.
- 필수 동의: 전체 동의와 함께 서비스 이용약관, 개인정보 수집·이용, 만 14세 이상 확인. 마케팅 수신 동의는 받지 않습니다. 각 항목에는 펼침(details) 형태의 약관·개인정보 초안 요약을 제공합니다.
- **개인정보 무저장 원칙**: 입력값(이름·전화·이메일·비밀번호·기관 정보 등 PII)은 React 컴포넌트 상태로만 존재하며 `localStorage`·`sessionStorage`·쿠키 어디에도 저장하지 않습니다. 프론트엔드 전용 완료 또는 초기화 시 모든 입력과 동의를 즉시 비웁니다.
- 완료 화면은 `가입 양식 확인 완료`만 표시하고 실제 계정이 생성되지 않았음을 명확히 안내하며, 요약에는 선택한 회원 유형과 `저장된 개인정보 없음`만 남깁니다.
- 순수 로직(초기 draft 생성·필드/전체 검증·휴대폰 포맷·전체 동의 토글·`clearDraftFields` 초기화)은 `src/signupFields.js`에 있으며 `src/signupFields.test.js`로 검증합니다. 완료·초기화 시 컴포넌트는 `clearDraftFields`로 회원 유형만 남기고 모든 PII·동의를 즉시 비웁니다.
- 이 폼은 **프론트엔드 양식 확인용**이며, 실제 플랫폼 인증·본인확인·서버 계정 생성은 아래 법무·개인정보 검토가 확정된 뒤에만 연결합니다. 기존 `SIGNUP_ENABLED=true` 경로(플랫폼 계정 인증 + `/api/account`)와 데이터 계약은 그대로 유지되며, 이 초안 폼은 비밀번호를 서버로 전송하지 않습니다.

## 최소 회원가입 구현 (SIGNUP_ENABLED=true 시)

- `/signup/doctor` 의료인 회원과 `/signup/hospital` 병원 회원의 가입 경로·목적·가입 후 이동을 분리합니다.
- 현재 Sites에서 실제 사용할 수 있는 플랫폼 계정 인증을 사용하며 서비스가 별도 비밀번호를 수집하지 않습니다.
- 가입 입력은 `의료인 / 병원·의료기관` 역할 하나이며, 전화번호·면허번호·주민등록번호·마케팅 동의를 받지 않습니다.
- 인증 이메일은 응답에서 현재 사용자에게만 표시하고 DB에는 이메일 원문 대신 서버 비밀키를 사용한 HMAC-SHA-256 값을 계정 키로 저장합니다.
- 이용약관, 만 14세 이상 확인, 개인정보 처리 안내 확인을 서로 다른 유형과 문서 버전으로 기록합니다.
- 소스에 포함된 약관·개인정보 안내 버전이 `draft`가 아니고 `SIGNUP_ENABLED=true`, `LEGAL_DOCUMENT_STATUS=approved`, 32자 이상의 `ACCOUNT_HASH_SECRET`가 모두 설정된 경우에만 서버 가입 API를 엽니다.
- Google·Kakao·Naver 외부 로그인은 Supabase 프로젝트와 각 사업자 앱 심사·키가 준비된 뒤 동일한 가입 완료 API에 연결합니다. 자격증명 없이 성공을 흉내 내는 버튼은 제공하지 않습니다.

### 의료인 회원

- 목적: 채용공고 탐색, 관심공고, 비공개 이직 상담, 지원 관리
- 가입 후 별도 확인: 비공개 공고 열람·소개 진행 시 의료인 면허 또는 자격
- 기본 이동: 채용정보

### 병원 회원

- 목적: 채용공고 등록, 채용 의뢰, 익명 인재 탐색, 동의 기반 소개 요청
- 가입 후 별도 확인: 공고 등록·소개 요청 시 의료기관과 담당자 관계
- 기본 이동: 인재정보

## 권한 정책

- 무료 권한은 공개 공고와 익명 인재의 기본 조건만 제공합니다.
- 결제 완료 시 상품별 `entitlement`를 발급하고 만료일·잔여 횟수를 검사합니다.
- 병원 멤버십은 익명 상세정보와 소개 요청권이며, 의료인의 연락처 구매권이 아닙니다.
- 의료인의 신원 정보는 후보자의 명시적 동의를 확인한 소개 건에서만 필요한 범위로 공개합니다.
- 민감정보 열람, 다운로드, 소개, 권한 변경은 모두 감사 로그에 기록합니다.
