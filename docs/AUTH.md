# AUTH

## Purpose

의료인·병원·운영자의 계정과 공개 범위를 정의합니다.

## Current State

현재 공개 사이트에는 실제 로그인이 없습니다. 다음 단계에서 역할 기반 계정을 도입합니다.

## Roles

- Doctor: 본인 프로필, 이력서, 지원·상담 내역 관리
- Hospital: 병원 인증, 공고, 광고 주문, 지원자 열람
- Headhunter: 상담, 매칭, 후보 추천, 상태 변경
- Admin: 회원·결제·게시물·권한 전체 관리

## Verification

- 의료인: 휴대폰 본인확인 + 면허 증빙 검토
- 병원: 사업자등록증 + 담당자 확인
- 운영자: 초대 방식과 다중 인증 권장

## Current Rules

- 의사 프로필은 기본 비공개입니다.
- 병원에는 의료인이 동의한 범위만 공개합니다.
- 광고 결제만으로 비공개 인재 연락처를 열람할 수 없습니다.
