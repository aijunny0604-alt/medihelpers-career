# 상담 시스템

## 사용자 흐름

1. 의료인 또는 병원을 선택합니다.
2. 상담 목적을 하나만 선택합니다.
3. 진료과, 지역, 근무·채용 형태 등 필수조건을 입력합니다.
4. 전화 또는 문자와 선호 연락시간을 선택합니다.
5. 입력 내용을 요약 확인하고 개인정보 수집에 동의합니다.
6. 상담번호를 발급하고 `신규 → 검토 → 연락 → 매칭 → 완료` 상태로 관리합니다.

## 상담 분류

- 의료인: 이직 가능성, 비공개 포지션, 급여·근무조건, 특정 공고
- 병원: 채용공고, 의료인 추천, 급여·채용조건 설계, 긴급 채용

## 운영 원칙

- 한 화면에서는 한 가지 결정만 요청합니다.
- 필수 입력은 진료과, 이름 또는 병원명, 연락처로 최소화합니다.
- 의료인의 구직 의사와 신원은 명시적 동의 전까지 병원에 공개하지 않습니다.
- 상담 목적과 조건에 따라 담당자, 우선순위, 후속 업무를 자동 분류합니다.
- 상담 접수·열람·상태변경·연락 이력은 운영자 감사 로그에 남깁니다.

## 현재 상태와 다음 단계

현재 공개 화면은 단계형 상담과 기기 내 임시 접수를 지원합니다. 실제 운영 전에는 서버 데이터베이스, 담당자 알림, 운영자 상담함, 접근권한과 개인정보 보유기간 정책을 연결해야 합니다.

전화 상담이 부담스러운 의료인을 위해 비동기 1:1 비공개 상담함을 후속 구현합니다. 첫 접수는 이름·현재 병원 없이도 시작할 수 있게 하고 `문자로 먼저 / 이메일 / 카카오톡 / 전화` 중 연락 방식을 선택하게 합니다. 공개 또는 익명 게시판 대신 상담자와 의사 헤드헌터만 대화를 열람하도록 하며, 병원 전달 전 별도 동의를 확인합니다. 세부 구현 체크리스트는 `ROADMAP.md`의 Phase 2 TODO를 기준으로 합니다.

매칭 리포트 상담에는 비교한 병원·후보, 사용자가 선택한 중요 조건, 확인 요청 질문을 함께 저장합니다. 현재 시범 운영은 기기 내 임시 저장과 이메일·전화 연결을 제공하며 담당자 자동 알림은 서버 상담함 구축 시 연결합니다.
