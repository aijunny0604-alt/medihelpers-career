# BILLING

## 수익 우선순위 (2026-07-18 갱신)

1. **병원 인재 이력서 열람권** — 주 수익. 병원이 인재 연락처·이력서 상세를 열람하려면 결제(1명 33,000원 / 5명팩 132,000원, 90일). [[TALENT_UNLOCK_PLAN]]
2. 병원 의사 헤드헌팅 성공보수 (아빠 직접 매칭, 계약 기반 오프라인)
3. 병원 의사 초빙공고 광고 (베이직 59,000 / 추천 149,000 / 집중 299,000원)
4. 의료인 채용공고 등록비 (미구현)
5. 선택형 의사 커리어 서비스 (알림·분석 등, 재정의 검토)

### 정책 원칙 (메디게이트 방식)
- **의사 급여·공고 상세는 로그인 회원이면 무료 열람** (멤버십 결제 불필요). 의사 회원을 최대한 모아 인재풀·병원 유입을 키우는 게 우선.
- **후보 개인정보(실명·연락처)는 상품으로 직접 판매하지 않는다.** 병원이 사는 것은 "열람권"이며, 후보 동의 범위에서만 열람. 인재정보 목록은 항상 익명.
- 헤드헌터 상담은 무료.

## 현재 구현 (2026-07-18)

- 서버 상품 카탈로그(광고 3종·멤버십 2종·**인재 열람권 2종**)로 상품명·금액 확정
- 병원/일반 회원 역할별 신청 가능 상품 검사, 주문 원장(주문·거래·영수증·환불·웹훅 스키마)
- **가상(테스트) 결제 모드**: 이니시스 키 미설정 시 결제 흐름을 시뮬레이션(주문→paid→거래기록 실제 작동, 실제 청구만 없음). 키 설정 시 자동으로 실제 이니시스 승인으로 전환.
- **이니시스 승인 API 골격**: `POST /api/payment-approve`(금액 위변조 검증), 웹표준결제 파라미터 생성. INICIS_MID/SIGN_KEY 설정 시 작동.
- **결제 라이프사이클 자동화**: 광고 결제완료→노출기간 부여, 환불/취소→노출 회수. 병원 마이페이지 '내 공고' 이용현황, 관리자 결제화면 노출기간 표시.
- **인재 열람권**: 병원 결제→`talent_unlocks` 권한 기록→`GET /api/talent-detail/:id`가 그 병원에만 연락처·이력서 상세 제공(감사 로그). 열람 없으면 잠금+구매버튼.
- 회원 마이페이지 주문 이력(실제 집계), 관리자 통합 콘솔·감사 로그.

## 아직 실제 결제가 아닌 부분

- PG 결제창 호출과 승인 API
- 결제 웹훅 서명·금액·중복 검증
- 카드 취소와 실제 환불 실행
- 세금계산서·현금영수증 자동 발행
- 결제 완료에 따른 광고 자동 게시·만료
- 환불 완료에 따른 권한·노출 자동 회수

따라서 현재 `paid`, `cancelled`, `refund requested` 상태는 관리자 운영 시연과 수동 기록을 위한 값입니다. PG 거래 결과로 간주하면 안 됩니다.

## 운영 결제 전 필수 규칙

- 주문 금액과 상품 조건은 주문 생성 시점 스냅샷으로 보존합니다.
- PG 승인 금액과 서버 주문 금액이 다르면 승인하지 않습니다.
- 웹훅은 공급자 이벤트 ID를 멱등키로 사용합니다.
- 환불 합계는 원 결제액을 넘을 수 없습니다.
- 결제·취소·환불 상태 전이는 허용된 순서만 통과시킵니다.
- 성공보수와 광고비는 별도 계약·청구 기준을 사용합니다.
- 환불정책과 증빙 발급 기준은 법무·세무 검토 후 확정합니다.
