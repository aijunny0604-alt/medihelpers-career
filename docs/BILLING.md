# BILLING

## 수익 우선순위

1. 병원 의사 헤드헌팅 성공보수
2. 병원 의사 초빙공고 광고
3. 병원 브랜드·추천 노출
4. 의료인 채용공고
5. 선택형 커리어 컨시어지

의사의 핵심 공고 정보와 헤드헌터 상담은 무료를 원칙으로 합니다. 후보자의 개인정보 자체를 상품으로 판매하지 않습니다.

## 현재 구현

- 서버 상품 카탈로그로 상품명·금액 확정
- 병원/일반 회원 역할별 신청 가능 상품 검사
- 주문번호, 공급가, 부가세, 총액, 구매자, 결제수단, 상태 저장
- 거래·상태 이벤트, 영수증 레코드, 환불 요청, 웹훅 원장 스키마
- 회원 마이페이지 주문 이력
- 관리자 주문 상태·수동 거래정보·환불 요청·감사 로그

현재 공개 화면에서 가격표를 상시 노출하지 않으며 광고 신청 또는 광고센터의 상세 안내 단계에서만 조건을 제시합니다.

## 아직 실제 결제가 아닌 부분

- PG 결제창 호출과 승인 API
- 결제 웹훅 서명·금액·중복 검증
- 카드 취소와 실제 환불 실행
- 세금계산서·현금영수증 자동 발행
- 결제 완료에 따른 광고 자동 게시·만료
- 환불 완료에 따른 권한·노출 자동 회수

따라서 현재 `paid`, `cancelled`, `refund requested` 상태는 관리자 운영 시연과 수동 기록을 위한 값입니다. PG 거래 결과로 간주하면 안 됩니다.

## 운영 결제 전 필수 규칙

- 주문 금액과 상품 조건은 주문 생성 시점 스냅샷으로 보존합니다.
- PG 승인 금액과 서버 주문 금액이 다르면 승인하지 않습니다.
- 웹훅은 공급자 이벤트 ID를 멱등키로 사용합니다.
- 환불 합계는 원 결제액을 넘을 수 없습니다.
- 결제·취소·환불 상태 전이는 허용된 순서만 통과시킵니다.
- 성공보수와 광고비는 별도 계약·청구 기준을 사용합니다.
- 환불정책과 증빙 발급 기준은 법무·세무 검토 후 확정합니다.
